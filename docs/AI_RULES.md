# AI Engineering Rules

## Core Principles
- Do not over engineer
- Prefer minimal production-grade implementations
- Write modular and scalable code
- Keep functions short and single responsibility
- Prioritize readability
- Avoid premature abstractions

## Architecture Standards
- Feature-first folder structure
- Separate business logic from UI
- Reusable services/utilities only when needed
- Avoid tight coupling

## Coding Style
- Small readable functions
- Relevant naming conventions
- Minimal comments
- No large files

## Documentation
After every implementation:
- Update relevant docs
- Update architecture notes
- Update progress tracking
- Update migration notes if required

## Session Ending Requirements
Always provide:
- Summary of implementation
- Updated files
- Remaining tasks
- Topics developer should study
